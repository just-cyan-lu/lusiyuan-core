# Agent Working Rules

这份文件记录代码代理在本项目里长期要遵守的工作规则。

## 新功能边界

新增功能前，先检查它是否和已有功能重叠。

- 如果能复用已有模块，就复用已有模块。
- 如果职责交叉，先切清边界，必要时清理旧入口。
- 如果只是换个名字把已有功能做一遍，通常不要做。
- admin 页面里的新能力要能被人看懂，不只写代码入口。

当前例子：

- `RuntimeEventDetail` 解释“发生了什么、有没有资格影响长期状态”。
- `StateChangeDetail` 解释“最终写入后，状态实际改了什么”。
- 两者共用 admin 详情展示组件，避免重复造 UI。
