# Database Tables

`prisma/migrations` is Prisma's database schema history. Each subdirectory is one
schema change. The numeric prefix is a timestamp used for ordering, and
`migration.sql` is the SQL Prisma applies to PostgreSQL.

This project is still in development, so the migration history is intentionally
kept compact. After the database model stabilizes, new production changes should
be appended as separate migrations instead of rewriting history.

| Prisma model | Database table | Purpose |
| --- | --- | --- |
| `User` | `app_users` | External identity for a person talking to Lusiyuan. |
| `Conversation` | `chat_conversations` | Chat session context for one user on one channel. |
| `Message` | `chat_messages` | Stored user, assistant, tool, and intermediate chat messages. |
| `ConversationContextSummary` | `conversation_context_summaries` | Compact summaries of older conversation ranges used as chat context. |
| `ChannelEvent` | `channel_events` | Raw inbound events received from Telegram, Weixin, or other channels. |
| `Memory` | `memories` | Long-term memories used to personalize future replies. |
| `ToolCallLog` | `tool_call_logs` | Audit trail for tool calls made by the assistant. |
| `SkillConfig` | `skill_configs` | Runtime configuration for optional skills. |
| `SystemSetting` | `system_settings` | Runtime settings stored in the database. |
| `SystemSettingEvent` | `system_setting_events` | Audit trail of runtime setting changes. |
| `XiaohongshuPost` | `xiaohongshu_posts` | Imported Xiaohongshu post snapshots. |
| `XiaohongshuComment` | `xiaohongshu_comments` | Comments imported from Xiaohongshu posts. |
| `XiaohongshuReplyDraft` | `xiaohongshu_reply_drafts` | Draft replies for Xiaohongshu comments. |
| `ExpressionLearningExample` | `expression_learning_examples` | Saved expression-learning examples. |
| `ExpressionLearningTrainingRecord` | `expression_learning_training_records` | Full records for expression-learning practice and teaching sessions. |
| `ExpressionLearningEmbedding` | `expression_learning_embeddings` | Vector embeddings for expression-learning examples. |
| `RuntimeState` | `runtime_states` | Current runtime state such as energy and activity summary. |
| `RuntimeStateEvent` | `runtime_state_events` | Compact audit events for runtime state changes. |
| `AutonomousTask` | `autonomous_tasks` | Idle-time autonomous tasks Lusiyuan can work on. |
| `AutonomousTaskRun` | `autonomous_task_runs` | Execution records for autonomous tasks. |
| `AutonomousArtifact` | `autonomous_artifacts` | Outputs created by autonomous task runs. |
| `PersonIdentity` | `person_identities` | A real-person identity that can contain multiple channel users. |
| `IdentityLink` | `identity_links` | Link between a channel user and a real-person identity. |
| `IdentityAlias` | `identity_aliases` | Self-claimed names for a real-person identity, independent of mutable channel display names. |
| `IdentityLinkProposal` | `identity_link_proposals` | Suggested links between channel users and identities. |

| `RelationshipStateEvent` | `relationship_state_events` | Audit trail for relationship state changes. |
| `RelationshipReviewProposal` | `relationship_review_proposals` | Pending or applied relationship profile reviews from Dream. |

| `MemoryEmbedding` | `memory_embeddings` | Vector embeddings attached to long-term memories. |
| `MessageEmbedding` | `message_embeddings` | Vector embeddings attached to chat messages for old-context recall. |
| `MemoryRiskFlag` | `memory_risk_flags` | Risk items detected by dream memory consolidation. |
| `GrowthLogProposal` | `growth_log_proposals` | Proposed growth-log style entries generated from dream consolidation. |
| `DreamJob` | `dream_jobs` | Dream cycle task that summarizes, extracts signals, and consolidates insights. |
| `DailyNote` | `dream_daily_notes` | Daily summary note generated by the dream cycle. |
| `DreamSignal` | `dream_signals` | Signal extracted from messages, memories, tools, or drafts. |
| `DreamDiaryEntry` | `dream_diary_entries` | Inner-diary style entry generated from dream signals. |
| `DreamConsolidationReport` | `dream_consolidation_reports` | Report for dream-cycle consolidation and generated memory changes. |
| `DreamLock` | `dream_locks` | Runtime lock that prevents overlapping dream jobs. |
| `ExternalPageSnapshot` | `external_page_snapshots` | Snapshot of an external page read by Jina, Playwright, or CDP. |
