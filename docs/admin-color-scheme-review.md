# Admin 配色方案 review

> 这份文档记录对陆思源 admin 平台现有配色的意见，以及本次颜色变量化之后的跟进状态。
> 原始 review 发生在 2026-06-25，随后完成了 admin 全量颜色变量迁移。

## 已完成的改动

本次已将 admin 组件中所有 6 位十六进制硬编码颜色迁移至 `web/src/index.css` 的 CSS 变量，涉及：

- `web/src/index.css`：新增/补齐 40+ 个 `--ls-*` 变量
- `web/src/components/admin/*.tsx`：21 个组件文件
- `web/src/components/admin/ANIMAL_ISLAND_UI_USAGE.md`：颜色参考表同步更新

迁移后 admin 组件里硬编码 hex 颜色为 0，`pnpm web:build` 通过。

## 原始意见与当前状态

### 1. 整体调色板方向正确，但冷暖对冲明显

**意见**：底色（奶油米白 + 暖棕文字）方向很好，但主色 `--ls-mint`（薄荷绿）和部分 accent（`--ls-blue`）偏冷，和暖黄、暖粉、暖棕放在一起时冷暖交界处会轻微打架。

**建议**：把薄荷绿往更偏「海泡绿 / 鼠尾草绿」微调（加一点黄），或把蓝色 accent 降权，只留给真正偏冷的模块（对话 / 梦境）。

**状态**：本次只做了变量化，没有改变色相。后续如果调整 `--ls-mint` 或 `--ls-blue` 的色值，因为现在全量用了变量，只需要改一处即可。

---

### 2. `--ls-ink-soft` 对比度偏低

**意见**：`--ls-ink-soft: #9f927d` 在 `--ls-panel: rgb(247, 243, 223)` 上的对比度约 2.8:1，作为说明文字长时间阅读会累，尤其是 10px、11px 的 nav label / subtitle。

**建议**：把 `--ls-ink-soft` 往深 5–8% 调，或给 subtitle/label 加 100 字重，不完全靠颜色区分层级。

**状态**：未调整。本次新增了 `--ls-ink-muted`、`--ls-ink-faint` 等中间层级，但没有改动 `--ls-ink-soft` 本身。

---

### 3. 背景层太满，信息层容易被「吃掉」

**意见**：`.admin-island-shell` 同时叠加了薄荷点阵、黄色点阵、三段渐变、底部两道径向光晕。每个 `Card` 又带自己的 `pattern`，背景 + 卡片都在发声，页面主体容易被背景纹理抢走注意力。

**建议**：让背景更轻（保留点阵但降低透明度 / 减少层数），把视觉重心交给卡片 pattern 和导航 accent。让后台更像「可爱的工具台」，不是 landing page。

**状态**：未调整。本次只替换了颜色值，没有改背景层的结构或透明度。

---

### 4. 硬编码颜色太多，变量系统半闲置 ✅ 已处理

**意见**：代码里大量 `text-[#794f27]`、`bg-[#e6f9f6]` 等硬编码，和已有的 `--ls-ink-strong`、`--ls-mint-soft` 重复。后续改一个主色要改几十处。

**建议**：补全 soft / light / text 变体变量，用 CSS 变量替代高频硬编码。

**状态**：已完成。admin 组件中 6 位 hex 颜色已清 0，全部使用 `var(--ls-*)`。

---

### 5. `StatusPill` 只有「启用 / 未启用」两种语义，粉色容易误读为错误

**意见**：`active ? 薄荷绿 : 粉色` 的粉色容易让人误以为「错误 / 危险」。admin 状态其实有更多层：启用、禁用、待审核、处理中、失败、警告。

**建议**：引入第三色（如 `--ls-yellow` 表示 pending/warning，`--ls-pink` 表示危险/失败），不要让粉色一直承担「未启用」这种中性状态。

**状态**：未调整语义映射。本次新增了 `--ls-red` 等变量，但 `StatusPill` 仍然只有两种状态色。后续如果要扩展状态，可以直接改 `StatusPill.tsx`。

---

### 6. `app-blue` 在语义上和其他模块区分不够

**意见**：规范说「对话 / 梦境：`app-blue`」，但 `AdminShell` 里「Active Model」状态卡也用了 `app-blue`。同时 `#889df0` 这个蓝在暖底上显得特别「跳」。

**建议**：把「Active Model / 配置 / 平台」这类偏系统模块统一到一个更中性的 accent（brown / yellow-green），保留蓝色给真正和「聊天 / 梦境」相关的部分。

**状态**：未调整。本次只把 `#889df0` 映射到 `--ls-blue`，没有重新分配语义。

---

### 7. 阴影色带灰紫调，和暖棕不太搭

**意见**：`--ls-shadow: 0 8px 24px rgba(61, 52, 40, 0.12)` 用的是冷灰棕，相比卡片边框的 `#e8dcc8` 偏冷。

**建议**：阴影往暖棕偏一点，例如 `rgba(91, 75, 55, 0.10)`。

**状态**：未调整。本次新增了 `--ls-3d-shadow`、`--ls-3d-shadow-strong` 并替换了相关硬编码，但没有改 `--ls-shadow` 的色值。

---

### 8. 禁用态颜色单独写死 ✅ 已处理

**意见**：`.field-input:disabled` 和按钮禁用态颜色是硬编码，且 `#ece8dc` 比 `--ls-panel` 更灰，在暖底上有点脏。

**建议**：增加 `--ls-disabled-bg` / `--ls-disabled-text`，保持禁用态也是暖的、不显得「褪色」。

**状态**：已完成。新增 `--ls-disabled-bg`、`--ls-disabled-text`，并替换了禁用态硬编码。

---

### 9. 移动端长标题配色没单独处理

**意见**：规范提到移动端不要用 `Title` ribbon，`DashboardPage` 里 fallback 的 `h2` 仍用 `--ls-ink-strong`。窄屏小尺寸下，深暖棕 + 粗黑体会显得更重。

**建议**：移动端标题可以考虑稍微提高明度（比如用 `--ls-ink-strong` 但字重降到 800），或者缩小字号时同步变浅一点。

**状态**：未调整。本次只做了颜色变量替换。

---

### 10. 全局长光标和整体配色和谐

**意见**：自定义 cursor 用了薄荷色小手指，和 `--ls-mint` 呼应，这个细节很好，可以继续保留。

**状态**：未改动，继续保持。

---

## 一句话总结

当前配色已经有个性、有风格，主要问题不是「颜色选得不好」，而是：

1. **变量系统没有真正落地** → ✅ 已解决
2. **背景层太满** → 待后续优化
3. **冷暖 accent 的权重和语义需要再分配** → 待后续优化

优先做的两件事（禁用态变量化、减少硬编码颜色）已经完成。后续如果要调色相、对比度、背景层复杂度，因为现在全量走变量，只需要改 `web/src/index.css` 即可。
