# 小红书评论发布设计

评论发布只面向从真实小红书页面读取并保存的帖子。它复用评论镜像、回复草稿和 `PlatformCommentPublishAttempt` 审计表，不单独维护第二套评论状态。

发布流程是：

1. Owner 读取已登录 Chrome 中的小红书帖子，生成并保存草稿；未保存的修改不能发布。
2. Owner 在后台点击“发布到小红书”并再次确认。
3. 服务重新定位帖子和目标评论，核验评论 ID、作者和正文；不一致即停止。
4. 服务只点击该条评论的回复入口、填写已确认草稿并提交。
5. 页面出现同内容的作者回复后，写入发布审计和账号镜像；页面无法确认时标为 `uncertain`，不自动重试。

`XIAOHONGSHU_COMMENT_PUBLISHER_ENABLED` 默认关闭。开启后也不改变逐条确认的要求。登录失效、验证码、风控提示、输入框或发布按钮无法可靠定位时，流程直接停止；不绕过验证，也不做批量发布。

发布本身不会启用表达学习。发布后由 Owner 依次点击“分析”和“学习”才会让该经验参与以后生成。
