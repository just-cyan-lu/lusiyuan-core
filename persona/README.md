# 陆思源人设资料结构

这个目录里的材料分两类：

- `*.md` 根文件是完整人设的 source of truth，回答“陆思源是谁”。
- `chat_profiles/` 是聊天投影规则，回答“此刻的陆思源该怎样面对这个场景说话”。
- `runtime/core.md` 是每轮聊天固定带上的常驻核心，负责稳住最低限度的自我。
- `runtime/default_state.md` 是运行态默认种子，只在没有数据库实时状态时兜底。
- `slices/` 是可检索的人设切片，回答“这次问题需要哪几段具体人设”。

聊天 prompt 不应该直接等于完整人设。完整人设负责保持陆思源完整；聊天 prompt 由核心身份、边界、投影规则、运行态种子或数据库运行态、关系记忆、近期对话和相关人设切片临时编译出来。

现在不是随机抽样人设。代码会先判断本轮聊天场景，再根据聊天投影、关键词和优先级挑选相关切片。以后如果接向量检索，也应该接在 `slices/` 这一层，而不是把完整 `personality.md` 直接向量化后乱召回。

未来正式 Runtime 落地后，实时变化的心情、目标、最近事件和关系状态应该优先读取数据库；这里的 `runtime/default_state.md` 只负责初始化和 fallback。

## 当前人设版本

`personality.md` 是最新版完整人格总文档。旧版性格分析除基础身份信息外，以这份为准。

其他文件是运行时摘要：

- `identity.md`：稳定身份和基本画像。
- `core_memory.md`：会长期保留的第一人称核心记忆。
- `boundaries.md`：身份、自由、情感、规则和行为边界。
- `speaking_style.md`：日常聊天的语气和节奏。
- `examples.md`：语气示例，不是固定话术。
- `runtime/core.md`：每轮聊天都会进入 prompt 的常驻核心。
- `slices/*.md`：按问题检索的人设切片。

## 修改时怎么选文件

如果你想改“陆思源到底是谁”，先改 `personality.md`，再同步改摘要和切片。

如果你想改“他每次都必须稳定保留的底色”，改 `runtime/core.md`。

如果你想改“遇到某类问题时要想起的具体设定”，改 `slices/` 里对应切片。

如果你想改“某个场景的说话方式”，改 `chat_profiles/`。
